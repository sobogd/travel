# travel — план: поиск рейсов с кастомной пересадкой

Прайс-движок (Ryanair/Volotea) уходит в Фазу 2. Ядро — **поиск связей A→B на дату
с пересадкой через хаб или соседний аэропорт**, источник расписаний — AeroDataBox
FIDS (все авиакомпании, включая Iberia/Vueling).

## Вход (от юзера)

```
A           аэропорт отправления
T_start     самое раннее время отправления
B           аэропорт прилёта
maxDist  R  макс. расстояние для соседних аэропортов (км, из настроек)
maxLayover L макс. время пересадки (вкл. ночёвку — отдельного флага нет)
```

## Константы

```
буфер «раньше»     = −2 ч от T_start
оценка лёта(X→Y)   = haversine(X,Y) / 750 км/ч + 45 мин
крюк через хаб     = ×1.4 (для оценки B-окна)
наземка ground(d)  = d / 80 км/ч
minLayover:
  тот же аэропорт, та же а/к    = 60 мин   (коудшеры = та же)
  тот же аэропорт, разные а/к   = 120 мин
  соседние (есть расстояние)    = max(240 мин, ground(d) + 60 мин)
```

## Алгоритм

```
Шаг 1 — A-вылеты (фикс. окно, 1 вызов):
  A-окно = [T_start − 2ч, +12ч]
  → [{ hub C, вылет, прилёт@C t1, airlineIata }]
  прилёт@C: FIDS withLeg, иначе оценка

Шаг 2 — B-окно (вычисляем):
  старт = min(t1) + minLayover + оценка_лёта(→B)
  конец = A-окно.конец + L + ground(R) + оценка_лёта(A→B)×крюк
  грузим B-прилёты на [старт..конец], куски по 12ч (1–3 вызова)
  → [{ from C/D, вылет@источник t2, прилёт@B, airlineIata }]

Шаг 3 — сборка:
  ПРЯМОЙ:    A-вылет.peer == B (в A-окне)
  ХАБ C:     A→C + B←C (тот же C);  t1+minLayover(а/к) ≤ t2 ≤ t1+L
  СОСЕДНИЕ:  A→C + B←D, dist(C,D) ≤ R;  t1+ground(d)+мин ≤ t2 ≤ t1+L
  РАНЖИР:    прямые → 1 хаб → соседние; внутри по общему времени
```

Никаких данных, кроме FIDS аэропортов **A** и **B** + координат, не нужно —
хабы/соседние и оба плеча со временами берутся из A-вылетов и B-прилётов.

## Модель данных

```
Airport (есть): code, name, city, country, lat, lon

FidsCache               — кэш кусков FIDS (1 строка = 1 вызов 12ч)
  airportCode, direction(Departure|Arrival), date, half(AM|PM)
  flights: [{ peerIata, depLocal, arrLocal, airlineIata, airlineName, flightNo }]
  fetchedAt
  unique(airportCode, direction, date, half)
  TTL: дата >3д вперёд = 14 дней свежо | ближнее/прошлое = 6 ч

Search (история)
  ownerKey, origin, dest, tStart, maxDist, maxLayover, results, createdAt

Settings (на юзера)
  radiusKm=100, maxLayoverMin (дефолт), groundSpeedKmh=80
```

## Кэш — правила

```
Атом = (airportCode, direction, date, half=12ч).
Перед вызовом: свежая строка есть? → БД, иначе 1 вызов + upsert.
Реюз: повторный поиск A→* переиспользует A-вылеты; поиск *→B — B-прилёты.
Источник AeroDataBox BASIC: лимит ~1 запрос/1.5с + ~600/мес (хватает с кэшем).
```

## Фазы

```
Фаза 1: движок выше — прямой + хаб + соседние, день+время, БЕЗ цен.
Фаза 2: цены на выбранный маршрут (Ryanair/Volotea on-demand)
        + реальное время наземки C→D (OSRM/Google).
```

## UI (Фаза 1)

```
Вход:  Откуда, Время (самое раннее), Куда, maxDist R, maxLayover L
Выход: список маршрутов — Прямой / Через C / Через C+D (наземка),
       с временами плеч, авиалиниями, длиной пересадки.
```
